Chapter I — Foundations

Stations & Storage

Build modular space stations on a crossword-puzzle grid. Enclose empty cells with walls to create airtight storage bays that hold up to 64 units each.

1.1 — Grid System

The Crossword Grid

Every station begins with a Structure Core at coordinate (0, 0). Modules are placed on an infinite sparse grid, one cell at a time. The core cannot be removed — it is the heart of your station. Three rules govern placement:

Interactive — hover cells to preview placement
Rule 1 — Connectivity
Every new module must be cardinally adjacent to at least one existing module. All modules must trace a path back to the core.
Rule 2 — No 2×2 Blocks
You cannot fill a 2×2 area with modules. This "crossword" constraint ensures visual spacing and forces interesting layout decisions.
Rule 3 — Interior / Exterior
Some modules require specific placement. Docking Bays must face open space (exterior). Crew Quarters, Medical, Recreation, and Greenhouses must be fully surrounded (interior).

1.3 — Enclosed Regions

How Storage Works

The key mechanic: empty cells fully surrounded by Operational modules form enclosed storage regions. The engine detects enclosures via flood-fill from the grid exterior — any cell unreachable from outside is "enclosed." Each enclosed cell can hold up to 64 units of a single resource type.

Enclosed cells shown in teal — modules shown as solid blocks

Connected enclosed cells form a region identified by a unique ID. Foundries use these region IDs to know where to pull input materials from and where to deposit outputs. Regions update automatically when modules are built or removed.

Decompression Warning
If you remove a wall module that breaks an enclosure, all stored resources in the exposed cells are explosively ejected into orbit — a radial burst of loose items around the station. They become collectible floating resources that your citizens must gather back.
1.4 — Storage Filters

Controlling What Goes Where

Right-click any enclosed cell to open the Zone Filter popup. Filters apply to all connected cells in a storage region. You can restrict which resource types are accepted — organized by tier: Ores (0–9), Metals (10–29), and Parts (30–49).

✓
No Filter (Default)
Zone accepts any resource type. Resources deposited first-come, first-served.
◉
Partial Filter
Only listed resource types accepted. Ore-only or ingot-only zones focus logistics.
✕
Empty Filter
Accept nothing. Useful as a buffer — block deposits temporarily while redesigning.

The filter popup also provides zone sorting: sort by name (A-Z), stack size (largest first), or material category. This reorganizes resources within the zone cells without ejecting anything.

1.5 — Construction Pipeline

From Blueprint to Operational

When you place a module, it enters the Planned state — a translucent blueprint on the grid. The module exists in the layout but provides no stats and does not count as a wall for enclosure detection. From here, the construction pipeline has three distinct phases:

The Construction Frontier
Only Planned modules adjacent to at least one Operational module can receive resources. This creates a "frontier" that expands outward from the station core. You can place blueprints far ahead, but workers will only deliver materials to the frontier — preventing construction in disconnected areas.

Phase 1 — Planned: The blueprint is placed. The module's recipe defines what resources are needed (e.g., Crew Quarters requires 2× Steel, 3× Glass, 1× Aluminum Ingot). No resources have been delivered yet.

Phase 2 — Under Construction: Once on the frontier, constructor citizens begin delivering resources. The module tracks resources_delivered vs resources_required for each material type. Once all resources are delivered, workers perform construction work — tracked as work_done / work_required. Work amounts vary by module complexity (150 for structural, up to 1250 for Shield Generators).

Phase 3 — Operational: The module is fully built. It now provides its stats (housing, food production, shield strength, etc.), counts as a structural wall for enclosure detection, and triggers an enclosure recalculation — potentially creating new storage regions behind it.

Cancelling Blueprints
Click a Planned or Under Construction module to cancel it. The blueprint is removed from the grid. Any resources already delivered are lost — they were consumed during delivery. Cancelling a Planned module that hasn't received resources costs nothing. Enclosure is recalculated immediately after cancellation.

1.6 — Demolishing

Deconstruction Mechanics

Shift-click an Operational module (except the Core) to mark it for deconstruction. The module enters the MarkedForDeconstruction state — it remains structurally present (still counts as a wall, still blocks flood-fill) until deconstruction completes.

Citizens with the Build task enabled will perform deconstruction work. Deconstruction takes 40% of the original construction time — faster than building. Once complete:

Material Refund
Each resource line in the recipe rolls independently: 25–65% refund (random), rounded up via ceil(). Cheap modules still return something.
Ejection to Orbit
Refunded materials are ejected into orbit as collectible floating resources — same explosive decompression physics as storage breach.
Enclosure Update
After removal, enclosure is recalculated. If the removed module was a wall, any exposed storage cells eject their contents. Docking assignments are also validated.
Structural Cascade
Removing a wall module can break multiple enclosures simultaneously. All stored resources in newly-exposed cells are ejected at once — a chain reaction of decompression. Plan demolition carefully: remove interior modules before exterior walls to minimize losses.
Walkthrough — Your First Station

Step by Step

1
Start with the Core
Every new station spawns with a Structure Core at (0,0). This is your anchor point. Open the Station Management window to see the grid editor.
2
Draw Structural Walls
Select Structure (Orange) or Structure (White) from the module palette. Click and drag to draw lines of wall segments extending from the core. Build a rectangular perimeter — leave the interior empty.
3
Complete the Enclosure
Close all gaps in your perimeter. Once construction bots finish building every wall segment to Operational, the empty cells inside will turn teal — they are now enclosed storage.
4
Place Interior Modules
Now place Crew Quarters, Greenhouses, and Medical bays on the walls (they must have all 4 neighbors occupied or enclosed). These functional modules need interior placement.
5
Add Docking Ports
Extend short arms outward from your walls and cap them with Docking Bays. These need at least one open-space neighbor (exterior placement). Citizens dock here to rest and transfer cargo.
6
Set Storage Filters
Right-click an enclosed cell to set what resources the zone accepts. Create dedicated ore storage, ingot bays, and component warehouses for efficient logistics.
Chapter II — Command

Crew & Assignments

Commission citizens to stations and toggle task permissions. Each citizen autonomously pursues enabled tasks based on priority, skill level, and station needs.

2.1 — Station Commissioning

Assigning Citizens to Stations

Citizens must be commissioned to a specific station to work there. Unassigned citizens appear in the recruitment panel. Click + Name to assign them. Commissioned citizens show in the crew matrix — a grid of portraits with action rows beneath.

If your planet has multiple stations, you can transfer citizens between them using the "Ship" row in the crew panel. Click the station name badge to open a picker showing all stations on the planet.

Crew assignment matrix — portraits as columns, tasks as rows

2.3 — Skills & Progression

Growing Your Crew

Citizens gain experience by performing tasks. Each task maps to a specific skill. Skills have 10 levels, displayed as a 2×5 military rank insignia (pips) on the action button. Higher skill levels increase work speed and efficiency.

The crew panel displays four status bars per citizen: HP (health), EN (energy), FD (hunger), and WK (work progress). Alert thresholds trigger warning badges — Critical HP below 30%, Exhausted below 15% energy, Starving at 0% hunger. A crew-wide alert banner summarizes fleet health at a glance.

2.4 — Docking & Rest System

Docking Port Assignments

Each Docking Bay module has berthing capacity (typically 4 at 100% efficiency). Citizens are assigned to specific docking modules for rest. When a citizen needs to rest, they fly to their assigned docking port, dock their ship, and begin recovery — regenerating energy and consuming food to replenish hunger.

Auto-Assignment
When a citizen first needs rest, they are automatically assigned to an available docking port with remaining capacity. Assignments persist across rest cycles — the citizen returns to the same port each time.
Manual Reassignment
You can manually reassign citizens via the crew panel GUI. If a docking module is destroyed, all citizens assigned to it are automatically unassigned and will seek a new port on their next rest cycle.

What happens during rest: Docked citizens recover energy over time. If food is available in station storage (resource ID 50, produced by Greenhouses), the citizen eats to restore hunger. Citizens with low energy (<15%) become "Exhausted" and will automatically seek rest if no higher-priority work is available. Citizens with 0% hunger are "Starving" and lose health each tick.

Crew Quarters vs. Docking Bays
Docking Bays are where ships physically dock for rest and cargo transfer. Crew Quarters provide housing capacity — citizens are also assigned to specific crew quarters modules for long-term residence. A station needs both: Docking Bays for the ship interface and Crew Quarters for the population cap (12 citizens per module at 100% efficiency).
Capacity Planning
If all docking ports are full when a citizen needs rest, they cannot dock and will keep working until exhausted. Build enough Docking Bays for your crew — a good ratio is 1 Docking Bay per 3–4 citizens, since not all citizens rest simultaneously.
Walkthrough — Managing Your First Crew

Step by Step

1
Commission Your Citizens
Open the Station Management panel. Scroll to "Unassigned Citizens" and click each name to commission them. They'll appear in the crew matrix above.
2
Enable Core Tasks
For a new colony, enable Mine + Collect + Build on most citizens. This gets ore flowing, resources gathered, and modules constructed. Keep at least one citizen with Search enabled to find asteroids.
3
Specialize as You Grow
As citizens level up, specialize them. A citizen with Mining Lv.5 mines much faster than Lv.1 — disable their other tasks and let them focus. Enable Smelt on 1–2 citizens once your foundry is running.
4
Monitor Crew Health
Watch the alert banner. If you see "Exhausted" or "Hungry" badges, ensure your station has enough docking bays for rest and greenhouses producing food. Use the Rest button to force-rest overworked citizens.
Chapter III — Industry

Foundry & Refinery

Foundries are membrane modules that sit between two storage regions — pulling raw materials from the input side and depositing products on the output side.

3.1 — The Membrane Model

Foundries as Walls

A Foundry is placed as part of a wall dividing two enclosed storage regions. It has a configurable output direction — one cardinal side deposits finished products. All other adjacent enclosed cells become input sources. This creates a directional flow: raw ore on one side, refined ingots on the other.

Foundry membrane — ore flows in from the left, ingots deposit to the right

The foundry auto-detects recipes from whatever ore is present in its input region. Workers assigned to the Smelt task automatically operate nearby foundries — no manual GUI configuration needed. Place ore in the input bay, and citizens will smelt it into ingots on the output side.

Key Requirement
Both input and output cells must be enclosed (part of a storage region) for the foundry to function. If the output cell's enclosure breaks, production halts until walls are restored.
3.2 — Ore Smelting

The most common foundry operation is smelting raw ore into basic ingots. Each ore type has a dedicated recipe that yields 7 ingots per ore chunk. This 1:7 ratio is the foundation of your entire industrial chain.

3.3 — Production Modes

Controlling Output

Foundries support three production modes that determine how long they keep working:

Process Exact
Craft exactly N items, then stop. Good for one-off batches. Tracks completed vs target count.
Stops when done
Accumulate
Keep producing until the output stockpile reaches N units. Pauses (doesn't clear) when target met — resumes automatically if stock drops below.
Self-regulating
Indefinite
Never stop. Always starts another job when one completes. Runs until input materials are exhausted or manually cancelled.
Runs forever

Greenhouses also use the directional output system — with an output direction set, food is deposited directly into the adjacent enclosed storage region instead of waiting for harvesters to collect it manually. This is the preferred setup for efficient food production.

Walkthrough — Setting Up Your First Foundry

Step by Step

1
Build a Divided Bay
Create a rectangular enclosure with a wall running through the middle, dividing it into two bays. Leave one gap in the dividing wall for a Foundry module.
2
Place the Foundry
Select the Foundry module and place it in the gap. Set the output direction to point toward your desired output bay (the side where you want ingots to appear).
3
Filter Your Input Bay
Right-click a cell in the input bay (the non-output side) and set the filter to accept only ores. This prevents refined ingots from accidentally being stored back in the input.
4

In the crew panel, enable the Smelt (⚒) task on at least one citizen. They will automatically find the foundry, pick up ore from the input bay, and begin smelting.
5
Watch the Flow
Ore chunks deplete from the input bay while ingots accumulate in the output bay. The storage overlay shows stack counts on each cell. You've built a production line!
Chapter IV — Manufacturing

Crafting System

From asteroid ore to advanced reactor cores — a six-tier industrial chain. Each tier builds on the last, creating an expanding web of recipes and resource dependencies.

4.1 — Resource Tiers

The Industrial Chain

Resources are organized into six tiers. Each tier's products become inputs for the next, creating a dependency chain from raw asteroid ore to complex components and fuel.

Tier dependency flow — mined ore cascades upward through the chain
T0 Raw Ores
10 types mined from asteroids. Iron, Copper, Carbon are common. Uranium, Rare Earth are exotic. IDs 0–9.
T1 Basic Ingots
Smelted from ores at 1:7 ratio. Iron Ingot, Copper Ingot, Silicon Wafer, etc. IDs 10–19.
T2 Alloys
Combined ingots: Steel, Bronze, Titanium Alloy, Carbon Fiber. Produced by Foundries. IDs 20–29.
T3 Components
Complex assemblies from T1+T2: Circuit Boards, Processors, Reactor Cores. Produced by Component Labs. IDs 30–49.
T4 Consumables
Food produced by Greenhouses. Consumed by citizens during rest to replenish hunger. ID 50.
T5 Propellants
Oxidized Fuel produced by Fuel Processors. Used by thrusters for ship movement. ID 60.

4.3 — Advanced Manufacturing

Assembler & Component Lab

Beyond the basic Foundry, two specialized modules handle higher-tier production. Unlike Foundries which auto-detect recipes, these require manual recipe assignment since they handle multiple possible outputs.

⚙
Assembler
Produces outputs in the ID range 10–29 (ingots and alloys). Works like a Foundry but handles Tier 2 multi-input recipes that require manual selection. Same membrane model — input/output regions separated by the module.
Recipes: Tier 1–2
🔬
Component Lab
Produces outputs in the ID range 30–49 (components). The highest-tier manufacturing module. Creates electronics (Circuit Boards, Processors), power systems (Battery Cells, Reactor Cores), and mechanical parts (Thruster Assemblies).
Recipes: Tier 3

Component recipes require a mix of Tier 1 ingots and Tier 2 alloys as inputs. For example, a Reactor Core needs 2× Uranium Ingot + 3× Steel + 2× Titanium Alloy — spanning three different source materials. Plan your input storage filters accordingly.

Fuel Processing
The Fuel Processor module converts 3× Carbon Brick + 2× Aluminum Ingot + 1× Rare Earth Ingot into 5× Oxidized Fuel (30s). It uses the same directional membrane model as Foundries. Fuel is consumed by ship thrusters for orbital movement.
4.4 — Multi-Input Production Chains

Complex Manufacturing

Beyond simple ore→ingot smelting, the real manufacturing challenge is multi-input recipes — crafting that requires 2–3 different material types simultaneously. These recipes are handled by Assemblers (T1–T2 outputs) and Component Labs (T3 outputs), both of which require manual recipe assignment since they can't auto-detect from input like basic Foundries.

Key Difference from Foundries
Basic Foundries auto-detect recipes from whatever ore is in the input bay. Assemblers and Component Labs produce multiple possible outputs from overlapping input materials — a Circuit Board and a Processor both need Silicon Wafer, but with different secondary inputs. You must manually assign a recipe to tell the module what to build. The function requires_recipe_assignment() distinguishes these modules.

Planning a multi-input line: Consider building a Reactor Core (the most complex T3 component). It requires 2× Uranium Ingot + 3× Steel + 2× Titanium Alloy. Steel is itself a T2 alloy (2× Iron Ingot + 1× Carbon Brick), and Titanium Alloy needs (2× Titanium Ingot + 3× Aluminum Ingot). The full dependency tree reaches 4 different ores:

Walkthrough — Building a Full Production Chain

Step by Step

1
Mine Diverse Ores
Enable Search and Mine on multiple citizens. You need at least Iron, Carbon, Copper, and Aluminum flowing in. These four ores unlock the most critical alloy recipes.
2
Smelt Into Ingots
Set up foundry bays (Chapter III walkthrough). Each 1 ore yields 7 ingots. Stock up — alloy recipes consume 2–3 ingots per craft. You'll burn through ingots quickly.
3
Produce Steel & Alloys
Set a foundry's input bay filter to accept Iron Ingot and Carbon Brick. Steel (Iron + Carbon) is the most consumed alloy — needed for Structural Steel, components, and module construction.
4
Build a Component Lab
Once you have a steady alloy supply, construct a Component Lab module. Assign it a recipe (e.g., Circuit Board: 2× Silicon Wafer + 1× Copper Wire + 1× Glass). Ensure the input bay is filtered to exactly those materials.
5
Scale & Specialize
Build dedicated production lines: one foundry bay for Steel, another for Titanium Alloy. Use storage filters to separate outputs. The Crafting Database page (Crafting tab) has a calculator to plan exact material requirements.

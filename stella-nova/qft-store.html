Store data in the angles

The honest way to use spin states as memory: write your bytes straight into the rotation angles each qubit actually controls, then read them back. n qubits give 2n real dials (a θ and φ each); quantize each to B bits and you have a clean, exact 2nB-bit register. No fitting, no approximation, no loss — the data lives where the hardware lives.

What you're watching
You're storing bits in the degrees of freedom the device controls — 2 angles per qubit — and decoding them exactly. This is the opposite of the lossy approach where you try to cram a 2ⁿ-sample curve into 2n knobs (a lossy 2ⁿ→2n squeeze that can never be exact). Here we store exactly 2nB bits, matched to the 2n dials, and read them straight back. Capacity scales linearly with qubits, decode is trivial, fidelity is perfect. If you wanted to store a full 2ⁿ-amplitude vector instead, you'd need an entangling write circuit and full state tomography to read — exponential, and not what you want for a dummy data channel.

The catch that matters: the 2nB figure is a property of this classical simulator, which reads the float angles back perfectly. On a real device you can't read an angle — a measurement returns one bit per qubit and destroys the state, and no-cloning forbids copying it. Holevo's theorem (1973) caps what's retrievable from n qubits at exactly n classical bits, no matter how finely you slice the angles. So as a simulator data structure this is a fancy float array; as a claim about quantum storage, the ceiling is n bits.
